Engenharia de Software

O que é Engenharia de Software?

Engenharia de Software é a disciplina que se ocupa do desenvolvimento de software de forma sistemática, controlada e eficiente, na manutenção e gerenciamento de softwares.

O que é um software?

Software é um conjunto de instruções que dizem ao computador como realizar uma tarefa específica.

Quais são os tipos de software?

  • Software de sistema
  • Software aplicativo
  • Software de programação
  • Software embarcado

Importância da Engenharia de Software

A Engenharia de Software é importante pois garante a qualidade, segurança, confiabilidade e eficiência dos sistemas desenvolvidos, e impacta diretamente a produtividade e a inovação.

O que faz um Engenheiro de Software?

  • Desenvolve e mantém sistemas de software
  • Aplica princípios, técnicas e melhores práticas para garantir que os softwares atendam às necessidades do usuário
  • Projeta e implementa soluções de software
  • Realiza testes e validações de software
  • Gerencia projetos de software
  • Documenta processos e resultados
  • Acompanha os avanços tecnológicos
  • Gerencia projetos e áreas relacionadas à TI

Um Engenheiro de Software pode atuar em diversas áreas, como:

  • Desenvolvimento de software
  • Testes de software
  • Gerenciamento de projetos
  • Arquitetura de software
  • Consultoria em tecnologia da informação

Como surgiu a Engenharia de Software?

Aprofissão surgiu na década de 1960, quando a complexidade dos sistemas de software começou a aumentar, necessitando de alguém que cuidasse disso.

Foi criada numa tentativa de contornar a crise do software, pois com o avanço da tecnologia, os softwares também evouíram, precisando de mais atenção sobre o que era feito.

Quais são as etapas do processo de desenvolvimento de software?

  1. Levantamento de requisitos
  2. Análise de requisitos
  3. Projeto do software
  4. Implementação
  5. Testes
  6. Manutenção
  7. Documentação
  8. Gerenciamento de configuração
  9. Gerenciamento de mudanças
  10. Gerenciamento de riscos
  11. Gerenciamento de qualidade
  12. Gerenciamento de projetos
